# 服务器上的 Git / 分布式 Git / GitHub 总结

本章总结第4-6章的重点内容：Git 服务器协议、分布式协作模式和 GitHub 使用。

## 1. 服务器上的 Git（第四章）

### Git 协议

Git 支持四种传输协议：

| 协议 | 特点 | 适用场景 |
|------|------|----------|
| **本地协议** | 通过文件系统共享 | 团队共享同一文件系统 |
| **HTTP(S)** | 智能协议，支持授权 | 最流行，易于使用 |
| **SSH** | 安全，需要 SSH 访问 | 需要授权的读写访问 |
| **Git 协议** | 速度快，无授权 | 大型只读项目 |

### 本地协议

```bash
# 克隆本地仓库
git clone /srv/git/project.git

# 添加远程仓库
git remote add local_proj /srv/git/project.git
```

- 优点：简单，直接使用文件权限
- 缺点：需要共享文件系统，不方便多位置访问

### SSH 协议

```bash
# 克隆
git clone ssh://[user@]server/project.git

# 简写
git clone [user@]server:project.git
```

- 优点：安全、易架设、高效
- 缺点：不支持匿名访问

### HTTP(S) 协议

```bash
# 智能 HTTP（推荐）
git clone https://example.com/gitproject.git
```

- 优点：支持匿名读写、授权、加密、企业防火墙友好
- 缺点：配置略复杂

### 在服务器上搭建 Git

```bash
# 创建裸仓库
git init --bare /srv/git/project.git

# 设置权限
chown -R git:git /srv/git/project.git
```

### SSH 公钥

```bash
# 生成 SSH 密钥
ssh-keygen -t ed25519 -C "your.email@example.com"

# 查看公钥
cat ~/.ssh/id_ed25519.pub
```

### 第三方托管服务

常见选择：
- **GitHub** - 最大最流行
- **GitLab** - 开源自托管
- **Gitee** - 码云，国内访问快
- **Bitbucket** - 支持 Git 和 Mercurial

## 2. 分布式 Git（第五章）

### 分布式工作流程

#### 集中式工作流

```
    ┌─────────────┐
    │  中央服务器  │
    └──────┬──────┘
           │
     ┌─────┼─────┬─────┐
     ▼     ▼     ▼     ▼
   A      B      C     D  (开发者)
```

- 所有人推送到一个中央仓库
- 类似 Subversion 的使用方式

#### 集成管理者工作流

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│ 维护者  │◄───│ 公共仓库 │◄───│ 贡献者  │
└─────────┘    └─────────┘    └─────────┘
     │                                 │
     └─────────────────────────────────┘
           (通过 pull request 或 patch)
```

- 贡献者fork公共仓库
- 维护者整合所有提交

#### 主管与副主管工作流

```
         ┌─────────┐
         │  主管   │
         └────┬────┘
              │
    ┌────────┼────────┐
    ▼        ▼        ▼
┌───────┐┌───────┐┌───────┐
│ 副主管 ││ 副主管 ││ 副主管 │
└───┬───┘└───┬───┘└───┬───┘
    │        │        │
    ▼        ▼        ▼
  开发者   开发者   开发者
```

- 多层级管理
- 适合大型团队

### 向项目贡献

#### 贡献方式

1. **直接推送** - 适用于受信任的团队成员
2. **Fork + Pull Request** - 适用于开源项目
3. **邮件补丁** - 传统方式，通过邮件发送 patch

#### 提交规范

```bash
# 清理提交历史
git rebase -i

# 创建有意义的提交
git commit -m "feat: add user authentication"

# 检查更改
git diff --stat
```

#### 使用 Pull Request

```bash
# 1. Fork 仓库
# 2. 克隆自己的 fork
git clone https://github.com/yourname/project.git

# 3. 添加上游仓库
git remote add upstream https://github.com/original/project.git

# 4. 创建功能分支
git checkout -b feature/my-feature

# 5. 开发并提交
git add .
git commit -m "Add new feature"

# 6. 推送
git push origin feature/my-feature

# 7. 在 GitHub 创建 Pull Request
```

### 维护项目

#### 应用补丁

```bash
# 应用补丁
git apply /path/to/patch.patch

# 检查补丁能否应用
git apply --check /path/to/patch.patch
```

#### 合并贡献

```bash
# 添加贡献者的仓库
git remote add contributor https://github.com/contributor/project.git

# 获取更新
git fetch contributor

# 审查更改
git log contributor/feature-branch

# 合并
git merge contributor/feature-branch

# 使用 --no-ff 保持分支历史
git merge --no-ff contributor/feature-branch
```

#### 打标签发布

```bash
# 创建发布标签
git tag -a v1.0.0 -m "Version 1.0.0"

# 推送标签
git push origin v1.0.0
```

## 3. GitHub（第六章）

### 账户配置

#### 创建账户

1. 访问 github.com
2. 注册用户名、邮箱、密码
3. 验证邮箱

#### SSH 密钥配置

```bash
# 生成密钥
ssh-keygen -t ed25519 -C "your@email.com"

# 添加到 SSH Agent
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# 在 GitHub 添加公钥
cat ~/.ssh/id_ed25519.pub
# 复制添加到 GitHub → Settings → SSH and GPG keys
```

#### 双因素认证

建议启用 2FA 提高安全性。

### 对项目做出贡献

#### Fork 工作流

```
原始仓库 → Fork → 本地修改 → Pull Request → 审核 → 合并
```

#### 创建 Pull Request

1. Fork 项目
2. 克隆到本地
3. 创建分支进行修改
4. 推送分支
5. 点击 "New Pull Request"
6. 填写描述并提交

#### PR 审核

- 查看文件变更
- 逐行评论
- 批准或请求修改
- 合并 PR

### 维护项目

#### 管理仓库设置

- 描述、网址
- 默认分支
- 协作者权限
- 分支保护规则

#### 分支保护

```bash
# 在 GitHub 界面设置：
# - 需要 PR 审核
# - 禁止强制推送
# - 需要状态检查通过
# - 要求代码所有者审核
```

#### 发布版本

1. 点击 "Releases" → "Draft a new release"
2. 选择标签
3. 填写发布说明
4. 上传资产文件

### 管理组织

#### 组织概念

```
组织 → 团队 → 成员 → 仓库
```

- **组织**：管理多个仓库和团队
- **团队**：按功能或项目分组
- **成员**：组织成员
- **仓库**：项目代码

#### 创建组织

1. 点击 "+" → "New organization"
2. 选择免费计划
3. 输入组织名称
4. 邀请成员

#### 团队权限

- **Owner**：完全控制
- **Admin**：管理仓库设置
- **Write**：推送和合并
- **Read**：读取和克隆

### 脚本 GitHub（GitHub CLI）

#### 安装

```bash
# macOS
brew install gh

# 其他系统
# 参见 https://github.com/cli/cli
```

#### 常用命令

```bash
# 登录
gh auth login

# 克隆仓库
gh repo clone owner/repo

# 创建 PR
gh pr create --title "Add feature" --body "Description"

# 查看 PR
gh pr list
gh pr view 123

# 合并 PR
gh pr merge 123

# 创建 release
gh release create v1.0.0 --notes "Release notes"

# 查看状态
gh status
```

#### 自动化

```bash
# 触发 Actions
gh workflow run build.yml

# 查看 run
gh run list
```

## 4. 总结

### 服务器部署要点

1. 选择合适的协议（HTTP/S 或 SSH）
2. 创建裸仓库 `--bare`
3. 配置 SSH 公钥认证
4. 考虑使用第三方托管服务简化运维

### 分布式协作要点

1. 选择适合团队规模的工作流
2. 建立提交规范（conventional commits）
3. 使用 Pull Request 进行代码审核
4. 定期同步上游仓库

### GitHub 使用要点

1. 启用 SSH 密钥和 2FA
2. 使用 Fork + PR 贡献开源
3. 利用组织管理团队
4. 使用 GitHub CLI 提高效率
5. 配置分支保护规则

### 推荐工作流

```
1. Fork 仓库（开源）或 克隆（内部）
2. 添加上游仓库
3. 创建功能分支
4. 开发并提交
5. 推送并创建 PR
6. 等待审核
7. 合并后删除分支
8. 同步上游更新
