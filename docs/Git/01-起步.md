# Git 起步总结

本章为 Git 入门，介绍版本控制工具的背景知识以及如何在系统上运行 Git。    

本部分参考自 https://git-scm.com/book/zh/v2 ，由AI进行总结提取。

## 1. 关于版本控制

版本控制是一种记录一个或若干文件内容变化，以便将来查阅特定版本修订情况的系统。

### 本地版本控制系统

采用某种简单的数据库来记录文件的历次更新差异。最流行的是 RCS，工作原理是在硬盘上保存补丁集（文件修订前后的变化）。

### 集中化的版本控制系统（CVCS）

如 CVS、Subversion、Perforce，有一个单一的集中管理的服务器，保存所有文件的修订版本。

- **优点**：管理员可以轻松掌控每个开发者的权限
- **缺点**：中央服务器的单点故障，宕机期间无法协同工作

### 分布式版本控制系统（DVCS）

如 Git、Mercurial、Darcs，客户端把代码仓库完整地镜像下来，包括完整的历史记录。

- **优点**：任何一处协同工作用的服务器发生故障，都可以恢复
- 可以指定和若干不同的远端代码仓库进行交互

## 2. Git 简史

Git 诞生于 2005 年，由 Linux 内核之父 Linus Torvalds 开发。起因是 BitKeeper 商业公司收回了 Linux 内核社区免费使用 BitKeeper 的权力。

Git 的设计目标：
- **速度** - 极快的性能
- **简单的设计** - 易于理解和使用
- **对非线性开发模式的强力支持** - 允许成千上万个并行开发的分支
- **完全分布式** - 不依赖中央服务器
- **有能力高效管理超大规模项目** - 速度和 数据量

## 3. Git 是什么？

Git 保存的不是文件的变化或者差异，而是一系列不同时刻的**快照**。

### 核心特性

1. **直接记录快照，而非差异** - 每次提交保存项目完整快照
2. **几乎所有操作都是本地执行** - 无需网络连接
3. **Git 保证完整性** - 所有数据在存储前都会计算校验和（SHA-1 哈希），之后以校验和来引用
4. **Git 一般只添加数据** - 很难让 Git 执行任何不可逆操作

### 三种状态

- **已提交（committed）**：数据已经安全地保存在本地数据库中
- **已修改（modified）**：修改了文件，但还没有保存到数据库中
- **已暂存（staged）**：对一个已修改文件的当前版本做了标记，使之包含在下次提交的快照中

### 三个工作区域

- **工作目录**：项目某个版本的单次检出
- **暂存区域**：保存下次将提交的文件列表信息
- **Git 仓库目录**：保存项目的元数据和对象数据库

## 4. 命令行

Git 有多种使用方式：
- 命令行工具（最完整的功能）
- GUI 工具（如 Git GUI、Gitk）

本书主要使用命令行。

## 5. 安装 Git

### 在 Linux 上安装

```bash
# Debian/Ubuntu
sudo apt-get install git

# CentOS/RHEL
sudo yum install git
```

### 在 macOS 上安装

```bash
# 使用 Homebrew
brew install git

# 或安装 Xcode Command Line Tools
xcode-select --install
```

### 在 Windows 上安装

从官网下载安装程序：`https://git-scm.com/download/win`

## 6. 初次运行 Git 前的配置

### 配置用户信息

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

### 配置文本编辑器

```bash
git config --global core.editor vim
```

### 检查配置

```bash
git config --list
git config user.name  # 查看单项配置
```

### 获取帮助

```bash
git help <verb>
git <verb> --help
man git-<verb>
```

## 7. 总结

Git 是一个分布式版本控制系统，具有以下特点：
- 速度快
- 简单的设计
- 强大的分支管理
- 完全分布式
- 能够高效管理超大规模项目

安装 Git 后，需要配置用户信息，然后就可以开始使用 Git 进行版本控制了。
